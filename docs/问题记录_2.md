# 项目问题记录（二）

---

## 1. `ModuleNotFoundError: No module named 'utils'`

**问题描述**：`from utils.config_handler import chroma_conf` 报错，但 PyCharm 能正常运行。

**原因**：Python 运行时不会自动将项目根目录加入 `sys.path`，PyCharm 会在运行配置中自动添加，而标准 Python 运行脚本时只把脚本所在目录加入 `sys.path`。

**解决方案**：创建 `utils/__init__.py`（空文件），使 `utils` 成为一个正式的 Python 包。

---

## 2. PyCharm 能运行但命令行报 `ModuleNotFoundError: No module named 'utils'`

**问题描述**：同一份代码在 PyCharm 中正常运行，在命令行运行时报错找不到模块。

**原因**：PyCharm 会自动将项目根目录加入 `PYTHONPATH`，但标准 Python 运行脚本时只把脚本所在目录加入 `sys.path`，不会自动添加项目根目录。

**解决方案**：使用 `pip install -e .` 将项目安装为可编辑包，使项目根目录始终在 Python 路径中。

---

## 3. `pip install -e .` 报错 `TOMLDecodeError: Invalid value`

**问题描述**：执行 `pip install -e .` 时报错 `TOMLDecodeError: Invalid value (at line 7, column 39)`。

**原因**：`pyproject.toml` 中 `include` 列表有语法错误，多了一个逗号：
```toml
include = ["utils*", "model*", "rag*",, "agent*"]  # 多余的逗号
```

**解决方案**：修正为 `include = ["utils*", "model*", "rag*", "agent*"]`，然后重新执行 `pip install -e . --force-reinstall --no-deps`。

---

## 4. 已安装 `pip install -e .` 但仍报 `ModuleNotFoundError: No module named 'agent'`

**问题描述**：执行 `pip install -e .` 后，运行 `python g:/agent/agent/react_agent.py` 仍报错找不到 `agent` 模块。

**原因**：`agent` 目录缺少 `__init__.py`，且 `pip install -e .` 在 `__init__.py` 创建之前就运行了，导致包扫描未包含新建的 `agent` 目录。

**解决方案**：
1. 创建 `agent/__init__.py`（空文件）
2. 创建 `agent/tools/__init__.py`（空文件）
3. 执行 `pip install -e . --force-reinstall --no-deps` 强制重新安装以刷新包扫描记录

**标记**：操作状态待确认

## 5. 使用系统 Python 而非虚拟环境 Python

**问题描述**：终端输入 `python` 调用的是系统 Python（`C:/Python314`），导致 `ModuleNotFoundError: No module named 'langchain_chroma'`。

**原因**：Windows 环境变量 `PATH` 中系统 Python 排在 venv Python 前面。

**解决方案**：
1. 使用 venv 完整路径运行脚本：`g:/agent/venv/Scripts/python.exe g:/agent/rag/vector_store.py`
2. 在 VSCode `.vscode/settings.json` 中配置 `"python.defaultInterpreterPath": "g:/agent/venv/Scripts/python.exe"`
3. 创建 `run.bat` 脚本自动激活 venv 后运行

## 6. `RecursiveCharacterTextSplitter` 报 `TypeError`

**问题描述**：`TypeError: TextSplitter.__init__() got an unexpected keyword argument 'separator'`。

**原因**：参数名错误，`RecursiveCharacterTextSplitter` 的参数名是 `separators`（复数），不是 `separator`。

**解决方案**：修正 `vector_store.py` 中的 `separator=chroma_conf["separators"]` 为 `separators=chroma_conf["separators"]`。

---

## 7. 向量库检索结果为 0 条

**问题描述**：检索词（如"扫地机器人"、"故障"）均无返回结果。

**原因**：`check_md5_hex` 函数调用逻辑错误，写成了 `if check_md5_hex:`（直接判断函数真假值），应该是 `if check_md5_hex(md5_hex):`（调用函数并传入参数）。导致所有文件均被判定为"已处理过"而跳过加载。

**解决方案**：修正为正确的函数调用写法：`if check_md5_hex(md5_hex):`。

---

## 8. `FileNotFoundError: md5.txt`

**问题描述**：代码尝试打开 `G:\agent\md5.txt` 时报文件不存在。

**原因**：
1. 代码中使用 `chroma_conf["md5.txt"]`，但配置文件 `chroma.yml` 中的 key 是 `md5_hex_store`
2. 初始文件不存在时用 `mode="r"` 尝试打开

**解决方案**：
1. 统一将 `md5.txt` 改为 `md5_hex_store` 读取配置
2. 修改为 `mode="w"` 创建空文件

---

## 9. Windows 控制台中文输出乱码（看起来像"无输出"）

**问题描述**：脚本运行后终端无任何输出（或显示乱码），实际上程序正常执行但中文内容在 Windows GBK 控制台显示为乱码。

**原因**：Windows 控制台默认使用 GBK 编码，而脚本输出为 UTF-8 编码的中文内容。

**解决方案**：运行 Python 时加 `-X utf8` 参数：
```bash
python -X utf8 g:/agent/rag/vector_store.py
```
或设置环境变量 `PYTHONIOENCODING=utf-8`。

---

## 10. 文本分割后序号不完整

**问题描述**：每段返回的字符串第一个序号缺失（如只有 `. 故障现象` 而非 `14. 故障现象`），且末尾仅有序号而无内容。

**原因**：
1. `chunk_size=200` 太小，同一个序号段落跨分割点时被截断
2. `separators` 列表中包含 `'.'`、`'?'`、`'!'` 等西文标点，会对中文序号（如 `14.`）进行误切割

**解决方案**：
1. 调大 `chunk_size` 至 400，`chunk_overlap` 调至 50
2. 调整 `separators` 列表，将 `""` 放在最前面作为兜底截断，中文标点靠后，西文标点移除
3. 删除 `chroma_db` 和 `md5.txt` 重新加载文档

## 11. PDF 加载后元数据包含大量不需要的字段

**问题描述**：PDF 的 `Document.metadata` 包含 `moddate`、`creator`、`producer`、`total_pages` 等大量元数据，只想知道 `source` 和 `page`。

**原因**：`PyPDFLoader` 默认将 PDF 的所有元数据全部写入 `Document.metadata`。

**解决方案**：在 `pdf_loader` 函数中加载后只保留需要的字段：
```python
doc.metadata = {"source": doc.metadata.get("source", file_path), "page": doc.metadata.get("page")}
```

## 12. `pip install -e .` 后出现 `agent_rag.egg-info` 文件夹

**问题描述**：项目目录中凭空出现 `agent_rag.egg-info` 文件夹，不确定是否可以删除。

**原因**：`pip install -e .`（可编辑模式安装）会由 setuptools 自动生成用于记录包的元数据。这是正常现象，不是错误。

**解决方案**：可以直接删除，不影响项目运行。

## 13. VSCode 右上角 "Run Code" 无反应

**问题描述**：点击 VSCode 右上角 "Run Code" 按钮无反应，但右键选择在终端中运行 Python 则正常。

**原因**："Run Code" 按钮使用的执行器由 VSCode Code Runner 插件控制，默认读取 `settings.json` 中的 `python` 配置，不使用选中的 venv Python 解释器。

**解决方案**：
1. 在 `.vscode/settings.json` 中配置 `"python.defaultInterpreterPath": "g:/agent/venv/Scripts/python.exe"`
2. 或直接使用终端运行脚本

## 14. `create_agent()` 传入 `system_prompt=load_system_prompt` 报错

**问题描述**：`TypeError: Unexpected keyword argument 'monitor_tool'` 后，改为 `model=chat_model` 后又报错 `ValidationError: Input should be a valid string [type=string_type, input_value=<function load_system_pro... at 0x...>]`。

**原因**：`load_system_prompt` 是函数对象本身（`<function>`），而 `create_agent` 需要的是**函数调用的返回值**（字符串）。

**解决方案**：改为 `system_prompt=load_system_prompt()`（调用函数获取字符串）。
