## 2. get_abs_path 函数路径基于问题

**问题描述**：`get_abs_path` 函数的路径是基于当前工作目录，不是全局

**解决方案**：修改为基于 `__file__` 所在目录计算绝对路径

```python
base_dir = os.path.dirname(os.path.dirname(os.path.abspath(__file__)))
return os.path.join(base_dir, relative_path)
```

## 4. Windows VSCode Code Runner 中文乱码

**问题描述**：Code Runner 输出面板使用系统默认编码（GBK），中文显示乱码

**尝试的解决方案**：
- 添加 `PYTHONIOENCODING: utf-8` 环境变量 → 对输出面板无效
- 设置 `code-runner.runInTerminal: true` → 改为输出到终端显示正常

**最终解决方案**：将中文日志/输出内容改为英文（`file_handler.py`、`md5_handler.py` 等）

---

## 5. Python 文件直接运行导入模块失败

**问题描述**：`python utils/config_handler.py` 报错 `No module named 'utils'`，但 PyCharm 能正常运行

**原因**：PyCharm 自动将项目根目录加入 `sys.path`，命令行直接运行则不会

**解决方案**：
1. ~~try/except fallback~~ → fallback 路径仍不正确
2. 动态添加项目根目录到 `sys.path` → 仅限当前项目结构
3. 创建 `.vscode/launch.json` 配置 `PYTHONPATH: ${workspaceFolder}` → F5 调试生效
4. 创建 `.vscode/settings.json` 配置 `terminal.integrated.env.windows PYTHONPATH` → 终端生效

**标记**：不确定最终方案是否完全生效，launch.json 和 settings.json 的修改状态待确认
